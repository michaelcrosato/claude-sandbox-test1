# Repo map

Where things live and what to skip. Pair with [AGENTS.md](../../AGENTS.md) (rules + loop).
Language: TypeScript/ESM, Node >= 22.13. Tests: Vitest, colocated `*.test.ts`. No linter/formatter
by choice — strict `tsc` is the style gate (see [AGENTS.md](../../AGENTS.md) › Conventions).

## Entry points

| Path | Role |
| --- | --- |
| `src/main.ts` | Process entry — CLI dispatch (`posthorn`, `posthorn admin`, `posthorn client`) + server bootstrap. |
| `src/index.ts` | Library public exports (`createGateway`, `loadConfig`, `PosthornClient`, `verifyWebhook`, store primitives). |
| `src/runtime/gateway.ts` | `createGateway` — wires stores, worker, dispatcher, HTTP server together. |
| `src/runtime/config.ts` | `loadConfig` — **authoritative** parsing/validation of every `POSTHORN_*` env var. |
| `src/version.ts` | Version constant (surfaced as the `posthorn_build_info` metric). |

## Core subsystems (`src/`)

| Dir | What |
| --- | --- |
| `signing/` | Standard Webhooks HMAC-SHA256 sign/verify, replay window, multi-token rotation. |
| `storage/` | Message stores (in-memory / SQLite / Postgres) + `conformance.ts`. |
| `queue/` | Durable leased delivery queue (in-memory / SQLite / Postgres) + retry / cancel / replay-endpoint / retry-app + `conformance.ts`. |
| `endpoints/` | Endpoint entity + stores + `endpoint-resolver.ts` (event-type/channel filter match) + `conformance.ts`. |
| `apps/` | Tenant (`App`) entity + API keys + stores. |
| `attempts/` | Per-attempt delivery audit records + stores. |
| `event-types/` | Event-type catalog (with `schemaExample`) + stores + `conformance.ts`. |
| `delivery/` | `retry-policy.ts` (jittered backoff), `delivery-state.ts` (state machine / dead-letter), `failure-reason.ts`. |
| `worker/` | `delivery-worker.ts` — claims leases, signs, POSTs, records attempts, folds endpoint health. |
| `fanout/` | Transactional outbox → per-endpoint deliveries (`fanout.ts`, `fanout-dispatcher.ts`). |
| `net/` | SSRF defense: `ssrf-guard.ts` (URL policy), `guarded-lookup.ts` (DNS-pin), `guarded-transport.ts` (connection-time guard + timeouts). |
| `http/` | `server.ts` (node:http + socket timeouts + shutdown drain), `router.ts`, `api.ts` (tenant + admin routes), `openapi.ts` (`GET /openapi.json`, drift-tested), `security-headers.ts`, `error-codes.ts` (closed `Error.code` set). |
| `billing/` | `billing-provider.ts` (interface) + Noop + `stripe-billing-provider.ts` + `stripe-signature.ts` (all behind flags; no live keys). |
| `dashboard/` | Operator + tenant browser dashboards (views / handlers / sessions). |
| `portal/` | Consumer (end-customer) portal — session / views / handler. |
| `pruner/` | `data-pruner.ts` — retention sweep. |
| `system-events/`, `logging/`, `metrics/`, `db/` | Internal system events · structured JSONL logger · Prometheus exposition · low-level SQLite/Postgres connection + pool. |
| `sdk/` | TypeScript SDK — `client.ts` (`PosthornClient`, `PosthornAdminClient`), `verify.ts` (`verifyWebhook`). |
| `runtime/` | `gateway.ts`, `config.ts`, `admin.ts` (admin CLI), `client-cli.ts` (client CLI), `backup.ts` (backup/restore). |

## Tests, contracts, config

- **Tests:** `src/**/*.test.ts` via `npm test`. Cross-backend: `src/{storage,queue,endpoints,apps,attempts,event-types}/conformance.ts` — extend when a store changes; Postgres skips unless `POSTHORN_TEST_PG_URL`.
- **Compiled-dist smokes:** `scripts/smoke-*.mjs` — `npm run build` first; they hit `127.0.0.1`.
- **Config:** `.env.example` (template, every var documented) · `src/runtime/config.ts` (authoritative) · `docs/DEPLOY.md` (ops runbook — new vars go here too).
- **Contracts:** `GET /openapi.json` ← `src/http/openapi.ts` (drift-tested) · TS SDK `src/sdk/` + Python SDK `clients/python/` · error codes `src/http/error-codes.ts`.

`Dockerfile`, `docker-compose.yml`, `deploy/helm/posthorn/`, `.github/workflows/ci.yml`

`*.db*` (SQLite data) · `**/__pycache__/` · `.git/` · `docs/log/` (LOG archives — read the
`docs/LOG.md` tail instead).
