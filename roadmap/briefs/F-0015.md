# F-0015 Brief: Prometheus Metrics Endpoint

## Feature

Expose unauthenticated operator metrics at `GET /metrics` in Prometheus text format so deployments can alert on ingestion, delivery outcomes, dead letters, uptime, and build info without reading tenant payload data.

## Acceptance

- `GET /metrics` returns Prometheus text exposition with content type `text/plain; version=0.0.4; charset=utf-8`.
- Non-GET `/metrics` returns `405 method_not_allowed` using the standard error envelope.
- Metrics are derived from durable SQLite tables so they observe gateway and worker activity.
- Metric labels are bounded and documented. They never include tenant IDs, app IDs, endpoint IDs, endpoint URLs, message IDs, event types, payload fields, headers, API keys, signing secrets, or hashes.
- Tests pin metric names, label sets, representative values, content type, trailing newline, and leakage guards.

## Metrics

- `posthorn_messages_ingested_total` counter, no labels: sum of accepted messages from `usage_months.messages_accepted`.
- `posthorn_deliveries_total` counter, `outcome="succeeded|retrying|dead_lettered"`: grouped from `delivery_attempts.outcome`.
- `posthorn_delivery_tasks` gauge, `status="pending|delivering|succeeded|dead_letter"`: grouped from `deliveries.status`.
- `posthorn_dead_letter_tasks` gauge, `reason="http_###|timeout|network_error|invalid_payload|signing_secret_unavailable|unknown|other"`: grouped from `deliveries.last_error` for `dead_letter` rows.
- `posthorn_uptime_seconds` gauge, no labels: elapsed seconds since gateway start.
- `posthorn_build_info` gauge, `version="0.0.0"`: static package/API version, value `1`.

## Implementation Plan

1. Add `src/metrics.ts` with a deterministic Prometheus text renderer and storage query helpers.
2. Track `startedAt` in `createGateway` and expose exact `GET /metrics` before auth routes.
3. Write metrics with plain-text content type; reuse JSON error envelope for `405` and storage-not-ready `503`.
4. Update `src/openapi.ts` so `/metrics` is an implemented unauthenticated route with `text/plain` success content.
5. Mark README `/metrics` implemented and update monitoring examples to real metric names/labels.
6. Add `tests/metrics-http.test.ts` using synthetic tenants/endpoints/messages and worker ticks for success, retrying failure, and dead-letter.
7. Run local checks, evaluator, and security review because this is an unauthenticated data route.

## Risks and Guards

- Scrape-time SQL counters can decrease if apps are deleted and cascaded. That is acceptable for this MVP; append-only metric counters can be added when strict monotonicity across deletes is required.
- Full-table grouping is acceptable at current scale. Add indexes later before high-volume deployment if scrape latency becomes material.
- Do not add a runtime Prometheus dependency; a small local formatter is sufficient and keeps the zero-runtime-dependency posture.
