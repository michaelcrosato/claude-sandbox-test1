# F-0010 Brief: Admin Tenant and API-Key Management

## Scope

Implement the disabled-by-default admin control plane described in `README.md#api-routes`. Operators need an admin token to provision tenants and manage tenant API keys before real producers can use the gateway.

## Acceptance Criteria

1. Admin routes return 404 unless POSTHORN_ADMIN_TOKEN is configured.
2. With a valid admin token, operators can create, list, read, update, and delete tenant apps.
3. Operators can mint, list, and revoke tenant API keys, with new phk_ secrets shown once.
4. Tests cover disabled admin surface, invalid admin token, app CRUD, key mint/list/revoke, and revoked-key authentication failure.

## Route Contract

- Admin auth uses `Authorization: Bearer <POSTHORN_ADMIN_TOKEN>`.
- When `adminToken` is unset/null, every `/v1/admin...` route returns `404 not_found`.
- When admin is enabled but the token is missing or wrong, return `401 unauthorized`.
- Unsupported methods on an existing enabled admin route return `405 method_not_allowed`.
- Validation failures return `400 invalid_request`.

Routes:

- `POST /v1/admin/apps`
  - Body: `{ "name": string, "monthlyMessageQuota"?: number | null }`.
  - Returns `201 { "app": AppRecord }`.
- `GET /v1/admin/apps`
  - Returns `200 { "data": AppRecord[] }`, newest first.
- `GET /v1/admin/apps/:id`
  - Returns `200 { "app": AppRecord }` or `404`.
- `PATCH /v1/admin/apps/:id`
  - Body may include `name` and/or `monthlyMessageQuota`.
  - Returns `200 { "app": AppRecord }` or `404`.
- `DELETE /v1/admin/apps/:id`
  - Deletes the tenant and cascaded rows through SQLite foreign keys.
  - Returns `204` or `404`.
- `POST /v1/admin/apps/:id/keys`
  - Body optional: `{ "name"?: string | null }`.
  - Returns `201 { "apiKey": ApiKeyRecord, "secret": "phk_..." }`.
  - The plaintext secret is shown only in this create response.
- `GET /v1/admin/apps/:id/keys`
  - Returns `200 { "data": ApiKeyRecord[] }` or `404` for a missing app.
  - Never returns plaintext secrets or key hashes.
- `DELETE /v1/admin/keys/:id`
  - Revokes a key by setting `revoked_at`; repeated revocation of an existing key may return `204`.
  - Returns `204` or `404`.

Record shapes:

- `AppRecord`: `{ id, name, monthlyMessageQuota, createdAt }`.
- `ApiKeyRecord`: `{ id, appId, name, revokedAt, createdAt }`.

## Implementation Notes

- Add a small `src/admin.ts` module for app and key operations.
- Use existing `apps` and `api_keys` tables; do not add schema unless a defect is found.
- Use `createApiKeySecret` and `hashApiKey` for tenant API keys.
- Use a constant-time admin token check by comparing SHA-256 digests of supplied and configured bearer tokens.
- Add `tests/admin-http.test.ts` for the full route surface and revocation behavior.
- Update exports from `src/index.ts` for admin primitives.
