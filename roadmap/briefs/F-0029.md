# Builder Brief — F-0029: Prometheus alerting and Grafana dashboard pack

> Immutable. Build exactly this; report back; nothing more. Explorer fan-out was skipped because this is a direct roadmap gap: `/metrics` and Docker Compose exist, but the promised Grafana dashboard / alerting rules do not.

## Feature

Add operator monitoring artifacts for the metrics Posthorn already emits.

This is a docs/artifact feature only. Do not change runtime metrics, source code, workflows, or deployment topology.

## Artifacts

Add:

- `docs/prometheus-alerts.yml`
- `docs/grafana-dashboard.json`

`docs/prometheus-alerts.yml` should define Prometheus rule groups for:

- Posthorn target down or missing (`up{job="posthorn"}`)
- Dead-letter backlog present (`posthorn_delivery_tasks{status="dead_letter"}`)
- New dead-lettered deliveries (`posthorn_deliveries_total{outcome="dead_lettered"}`)
- Stuck delivering backlog (`posthorn_delivery_tasks{status="delivering"}`)
- High retrying/failure signal (`posthorn_deliveries_total{outcome="retrying"}` and/or dead-lettered deliveries)

Rules must only reference metrics Posthorn actually emits:

- `posthorn_messages_ingested_total`
- `posthorn_deliveries_total`
- `posthorn_delivery_tasks`
- `posthorn_dead_letter_tasks`
- `posthorn_uptime_seconds`
- `posthorn_build_info`
- Prometheus scrape metric `up`

`docs/grafana-dashboard.json` should be an importable Grafana dashboard with:

- A Prometheus datasource variable named `DS_PROMETHEUS`.
- Panels for accepted messages, delivery outcomes, delivery task backlog by status, dead-letter reasons, uptime, and build info/version.
- Expressions using only implemented metric names above.
- No tenant-specific labels, sample secrets, localhost-only datasource URLs, or external service dependencies.
