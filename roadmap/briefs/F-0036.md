# F-0036 — CloudEvents delivery payload format

## Context

Posthorn now supports endpoint-level delivery method selection and two payload body shapes: the Posthorn envelope and the original payload only. The parity matrix still correctly marks arbitrary payload transformations as partial. This feature adds one more safe built-in format: CloudEvents JSON 1.0.

This is a closed declarative format. It is not arbitrary templates, JavaScript, JSONPath, body rewriting, URL rewriting, header mutation rules, content-type customization, arbitrary HTTP methods, or non-webhook connectors.

## Scope

- Extend endpoint `payloadFormat` to allow `cloud_events_1_0` in addition to `envelope` and `payload_only`.
- Preserve existing defaults:
  - New and legacy endpoints default to `envelope`.
  - Create with omitted or `null` `payloadFormat` uses `envelope`.
  - Update with omitted `payloadFormat` leaves the existing value unchanged.
  - Update with `null` resets to `envelope`.
- Real delivery with `cloud_events_1_0` sends this deterministic JSON object and signs that exact raw body:
  - `specversion`: `"1.0"`
  - `id`: message id
  - `type`: message event type
  - `source`: `"urn:posthorn"`
  - `time`: stored message creation time
  - `data`: original JSON payload
- Endpoint test-send with `cloud_events_1_0` uses the same shape with its test id, requested event type, current test time, and resolved payload.
- Preserve endpoint `deliveryMethod`, Standard Webhooks signatures, JSON content type, custom non-secret headers, manual redirects, retry behavior, tenant isolation, and SSRF URL validation.

## Acceptance

- Endpoint create/update validation and read/list serialization cover `cloud_events_1_0`, existing default/reset behavior, omitted unchanged behavior, and invalid values.
- Worker tests prove CloudEvents bodies contain the expected fields, use stored message time, preserve `deliveryMethod`, and verify Standard Webhooks signatures against the exact body.
- Endpoint test-send tests prove CloudEvents bodies contain the expected fields, use the test time, and still avoid durable message/delivery/usage/attempt rows.
- TS SDK, CLI, Python SDK, OpenAPI, README, Python README, and `docs/PARITY.md` expose `cloud_events_1_0` without claiming arbitrary transformations.
- Existing payload-only, envelope, delivery method, and signature tests continue to pass.
