# F-0035 — Endpoint delivery HTTP methods

## Context

Posthorn now supports endpoint-level payload formats, but the parity matrix still marks payload transformations as partial because delivery shape is intentionally limited. This feature adds one more safe declarative option: an endpoint can choose whether outbound webhook deliveries use `POST` or `PUT`.

This must stay a closed enum. It is not arbitrary transformations, request templating, custom content types, URL rewriting, custom secret headers, GET/DELETE/PATCH support, or non-webhook destination connectors.

## Scope

- Add `deliveryMethod` to endpoint storage, endpoint records, delivery targets, create/update/read/list API responses, OpenAPI, TypeScript SDK, tenant CLI, Python SDK, README, Python README, and `docs/PARITY.md`.
- Allowed values are exactly `POST` and `PUT`.
- New endpoints and migrated existing endpoints default to `POST`.
- Create with omitted or `null` `deliveryMethod` uses `POST`.
- Update with omitted `deliveryMethod` leaves the existing value unchanged.
- Update with `null` resets to `POST`.
- Invalid values return `invalid_request`.
- The worker uses the stored method while preserving current payload body selection, Standard Webhooks signing over the exact raw body, JSON content type, custom non-secret headers, manual redirect handling, retry behavior, tenant isolation, and SSRF URL validation.
- Endpoint test-send uses the same method/body/signature behavior as real delivery and still does not persist messages, deliveries, usage, or attempts.

## Acceptance

- Endpoint create/update validation and read/list serialization cover default `POST`, `PUT`, `null` reset, omitted unchanged, and invalid values.
- SQLite schema and legacy migration persist `delivery_method TEXT NOT NULL DEFAULT 'POST'` idempotently.
- Worker tests prove `PUT` is used, redirects remain manual, and the Standard Webhooks signature verifies against the exact selected raw body.
- Endpoint test-send tests prove it uses endpoint `deliveryMethod` with the same body/signature rules.
- TS SDK, CLI, Python SDK, OpenAPI, README, Python README, and parity docs expose the field without claiming arbitrary transformations.

## Explorer Notes

- Endpoint API/storage/client surfaces follow the existing `payloadFormat` pattern in `src/endpoints.ts`, `src/storage.ts`, `src/openapi.ts`, `src/client.ts`, `src/cli.ts`, `src/index.ts`, `clients/python/posthorn/client.py`, and their tests.
- Worker delivery currently hard-codes `method: 'POST'` in `src/worker.ts`; endpoint test-send does the same in `src/endpoint-tests.ts`.
- `signWebhook` signs raw body bytes only, so tests must pin exact body/signature behavior when the method changes.
- Assertion shield blocks deleted tests/assertions. Add coverage without removing existing assertions.
