# F-0019 Brief: Endpoint Signing Secret Rotation

## Feature

Implement zero-downtime endpoint signing secret rotation. Tenants should be able to rotate an endpoint's receiver verification secret, hand the new `whsec_...` value to their consumer, and keep deliveries verifiable with both the new and previous secret for a short overlap window.

## Product Context

The README already names zero-downtime secret rotation as a key Posthorn capability, and comparable webhook products such as Svix, Convoy, and Hookdeck surface rotation, replay, portal, and delivery-debugging flows as core reliability features. Posthorn has Standard Webhooks multi-signature support already; this feature makes endpoint rotation real at the API and delivery-worker layers.

## Acceptance

- `POST /v1/endpoints/:id/rotate-secret` returns `201 { endpoint, secret, previousSecretExpiresAt }`.
- The returned `secret` is a fresh `whsec_...` value shown exactly once and never included in later endpoint reads/lists.
- The endpoint row's current secret is replaced immediately for future deliveries.
- During the overlap window, delivery signatures include both current and previous endpoint secrets using the existing Standard Webhooks space-delimited multi-signature format.
- After the overlap window, only the current secret signs new deliveries.
- Rotation is tenant-scoped: another tenant gets `404`, missing/revoked auth gets `401`, invalid methods get `405`.
- Disabled endpoints may still rotate their secret; rotation is a credential-management action, not a delivery action.
- OpenAPI, README route rows, and tests mark the route implemented.

## Contract

Route:

- `POST /v1/endpoints/:id/rotate-secret`
- Auth: tenant bearer key.
- Request body: optional JSON object. For F-0019, accept `{ overlapSeconds?: number }`.
- Default overlap: 24 hours.
- Minimum overlap: 60 seconds.
- Maximum overlap: 30 days.
- Response:
  - `endpoint`: normal endpoint read model.
  - `secret`: new current signing secret, returned exactly once.
  - `previousSecretExpiresAt`: ISO timestamp for the previous secret's last usable delivery-signature time.

Storage:

- Preserve current endpoint secret columns for the active secret.
- Add nullable previous-secret columns to `endpoints`:
  - `previous_signing_secret_ciphertext`
  - `previous_signing_secret_key_version`
  - `previous_signing_secret_nonce`
  - `previous_signing_secret_expires_at`
- `initializeSchema` should add these columns idempotently for existing databases.

Delivery:

- When a previous secret exists and `previous_signing_secret_expires_at >= attemptedAt`, call `signWebhook([currentSecret, previousSecret], ...)`.
- Otherwise call `signWebhook(currentSecret, ...)`.
- If the previous secret cannot be revealed, fail closed with `signing_secret_unavailable` during its overlap; do not silently skip a corrupt configured rotation secret.

## Implementation Plan

1. Extend storage initialization with previous-secret columns and tests.
2. Add endpoint rotation domain helper in `src/endpoints.ts`.
3. Extend endpoint delivery target shape so the worker can see previous-secret metadata.
4. Wire `POST /v1/endpoints/:id/rotate-secret` in the gateway.
5. Add OpenAPI route and response schema.
6. Update README route status and SDK note.
7. Add HTTP tests for rotation response secrecy, tenant isolation, auth/method dispatch, disabled endpoint rotation, and overlap bounds.
8. Add worker tests proving multi-signature delivery during overlap and single-signature delivery after expiry.
9. Run fresh evaluator and security review because this touches credential storage and delivery signing.

## Risks and Guards

- Raw secret leakage: return the new secret once only; endpoint reads/lists, OpenAPI examples, logs, and evidence must not contain raw `whsec_` values except generic documentation placeholders.
- Backward compatibility: existing SQLite files need `ALTER TABLE ... ADD COLUMN` migrations that are safe to run repeatedly.
- Receiver cutover: default overlap should be long enough for customers to deploy the new secret but bounded so old secrets age out.
- Corrupt previous secret state: fail closed during active overlap rather than emitting only the new signature and silently breaking receivers still using the previous secret.
